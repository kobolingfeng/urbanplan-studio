# 城市更新智能规划助手毕业设计

本仓库用于推进一个硕士毕业设计方向：面向中国城市更新和详细规划的语义化规划建模、规则辅助审查与方案推演系统。

当前核心判断：

- 不从 CAD 插件开始，而从“规划对象模型 + 规则库 + 证据系统 + 方案推演引擎”开始。
- CAD/GIS/BIM 都可以作为输入、输出或外部底座，但内部核心应是可计算、可追溯、可比较的规划对象系统。
- 毕业设计应收敛到一个小而完整的原型：用户能够建立城市更新片区，绘制或导入地块，设置指标，运行规则检查，生成问题清单和诊断报告。

## 建议主线

推荐论文题目暂定为：

> 面向城市更新的语义化规划数据模型与规则辅助审查原型研究

这条主线足够聚焦，能把规划学问题、软件系统设计、数据模型、规则引擎和原型验证串起来。

## 第一阶段目标

第一阶段不追求“完整规划软件”，而是完成一个可展示、可解释、可验证的最小闭环：

1. 定义 UPF 0.1 最小对象集。
2. 建立 20 条左右可执行规则。
3. 构造一个城市更新片区样例数据。
4. 开发原型界面：地块编辑、指标计算、规则检查、报告输出。
5. 用样例验证规则结果是否可追溯、方案比较是否有解释力。

## 可交付成果

- 论文：研究背景、相关研究、数据模型、规则引擎、系统原型、案例验证。

- 数据：UPF 0.1 样例工程包。
- 规则：最小规则集与规则元数据。
- 报告：样例片区规划诊断报告。
