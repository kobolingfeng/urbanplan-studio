# UrbanPlan Studio 答辩演示脚本

本脚本用于硕士毕业设计答辩或中期检查时演示 UrbanPlan Studio。目标不是把系统包装成正式审批软件，而是清楚展示“语义化规划对象、规则检查、情景比较、证据链和报告输出”如何支撑城市更新早期方案推演。

## 1. 开场定位

建议开场表述：

> 本系统面向城市更新早期方案推演，不替代 CAD、ArcGIS 或法定审查平台。它把地块、道路、公共服务设施、出入口、开放空间、约束控制线和情景方案组织成 UPF 语义规划格式，再通过规则检查、综合评分、权重敏感性和数据质量诊断，帮助毕业设计阶段形成可复现、可解释、可比较的分析流程。

## 2. 演示顺序

1. 打开软件，说明三栏界面：
   - 左侧是项目、图层、对象和方案版本。
   - 中间是语义规划画布和规则/评估结果。
   - 右侧是对象 Inspector 和证据链。
2. 点击 A-01 地块，说明对象不是普通 CAD 图形，而是带有用地、控制指标、情景值和证据来源的语义对象。
3. 点击 `检查`，展示规则问题、综合评分、智能建议同步更新。
4. 切换方案列表，说明每个方案都有评分和状态，可快速识别优劣。
5. 点击公共设施对象，展示服务半径覆盖圈，说明公共服务覆盖从文字指标进入图面表达。
6. 点击 `评估`，展示六个维度的综合评分和地块优先级。
7. 点击 `敏感`，说明不同价值偏好下推荐方案是否稳定。
8. 点击 `对比`，展示所有方案的决策矩阵。
9. 点击 `质检`，展示证据覆盖率、规则目录和导入审计。
10. 点击 `验证`，展示研究问题、证据链、实验记录表、专家复核表和 CSV 附录。
11. 点击 `报告`，说明系统可直接导出 Markdown 作为论文附件或答辩材料。

## 3. 重点回答

### 3.1 为什么不是 CAD 插件？

CAD 适合绘图，但城市更新早期方案还需要处理对象属性、情景值、规则来源、证据链、评分模型和报告。UrbanPlan Studio 的研究重点是“规划对象数据化和可计算化”，因此用 UPF 作为中间格式，而不是继续围绕 CAD 图层扩展。

### 3.2 综合评分是不是主观？

当前评分是透明加权模型，不声称是绝对客观结论。系统提供四组权重画像进行敏感性分析：均衡模型、公共服务优先、保护与生态优先、实施风险优先。若推荐方案在多组权重下稳定，可以说明结果具有一定稳健性；若变化明显，则说明方案选择受价值偏好影响。

### 3.3 规则检查是否等于规范审查？

不是。系统输出是原型规则和辅助判断，必须保留来源、版本和限制说明。它适合用于毕业设计中的初筛、方案比较和问题定位，不替代法定控规审查、交通影响评价、消防审查或正式专项论证。

### 3.4 数据可信度如何保证？

系统要求每个对象记录证据来源，并在数据质量报告中显示证据覆盖率、规则目录、原型规则数量和导入审计。它不能保证原始数据真实完整，但能显式暴露数据缺口，避免“看起来精确但依据不明”。

7. 导出的规划诊断报告。

## 5. 演示风险

- 不要把评分说成正式审查结论。
- 不要承诺覆盖所有城市更新规范。
- 若被问到坐标精度，应说明当前 demo 使用米制画布比例，真实坐标/CRS 是下一阶段工作。
- 若被问到公共服务覆盖，应说明当前为原型覆盖表达，后续应接入路网步行距离。
